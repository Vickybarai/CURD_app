## 🔹 PHASE 1: CLOUD RESOURCE SETUP
*We set up the storage (S3) and database (RDS) first so we have their details ready for the server (EC2).*

### Step 1: Create S3 Bucket (Storage for Files)
We use S3 to store your application files so the EC2 server can download them easily.

1.  Go to **AWS Console** → **S3** → **Create bucket**.
2.  **Bucket name:** Enter a unique global name (e.g., `my-student-app-bucket-123`).
3.  **Region:** Select the same region where you will launch EC2 (e.g., **N. Virginia**).
4.  **ACLs:** Enable **ACLs enabled** (Check the box).
5.  **Block Public Access:** Uncheck **Block all public access** (Acknowledge the warning). *Note: This is for learning purposes to allow easy download.*
6.  Click **Create bucket**.

### Step 2: Upload Files to S3
1.  Open your newly created bucket.
2.  Click **Upload**.
3.  Drag and drop these two files from your computer:
    *   `Student.war`
    *   `mysql-connector-java.jar`
4.  Click **Upload**.
5.  **CRITICAL:** After uploading, click on `Student.war`, copy the **Object URL** (e.g., `https://bucket-name.s3.region.amazonaws.com/Student.war`), and save it in Notepad. Do the same for the `.jar` file.

### Step 3: Create RDS Database (MariaDB)
1.  Go to **AWS Console** → **RDS** → **Create database**.
2.  **Standard create** → **MariaDB**.
3.  **Template:** Free tier.
4.  **Settings:**
    *   DB Instance Identifier: `studentdb`
    *   Master username: `admin`
    *   Master password: `admin1234` (Note this down!)
5.  **Connectivity:**
    *   **Public access:** Yes (Required for direct connection from EC2 for this setup).
    *   **VPC Security Group:** Click **Create new** to create a specific Security Group for the DB.
6.  **Create database**.
7.  Wait for status to change to **Available** (approx 10-15 mins).
8.  **COPY:** Click the database, copy the **Endpoint** (e.g., `studentdb.cxyz.us-east-1.rds.amazonaws.com`).

### Step 4: Configure Security Groups (Networking)
We need to allow traffic to flow between your browser, EC2, and RDS.

1.  Go to **EC2 Console** → **Security Groups**.
2.  **Find the RDS Security Group** (Created in Step 3):
    *   **Edit Inbound Rules** → **Add Rule**.
    *   Type: **MySQL/Aurora** | Port: **3306** | Source: **Custom** -> Search and select the **EC2 Security Group** (if EC2 exists) or **Anywhere (0.0.0.0/0)** for testing.
3.  **Find/Create the EC2 Security Group**:
    *   **Edit Inbound Rules** → Add:
        *   **SSH (Port 22)**: Source: My IP.
        *   **HTTP (Port 80)**: Source: Anywhere (0.0.0.0/0).
        *   **Custom TCP (Port 8080)**: Source: Anywhere (0.0.0.0/0).

## 🔹 PHASE 2: EC2 SERVER SETUP
*Now we launch the server and install the required software.*

### Step 5: Launch EC2 Instance
1.  Go to **EC2 Console** → **Launch Instance**.
2.  **Name:** `App-Server`.
3.  **OS:** Amazon Linux 2023 (or Amazon Linux 2).
4.  **Instance Type:** `t3.micro` (Free Tier eligible).
5.  **Key Pair:** Create or select your Key Pair (keep `.pem` file safe).
6.  **Network Settings:**
    *   Select the **EC2 Security Group** we configured in Step 4.
    *   **Public IP:** Enable (Assign public IP automatically).
7.  **Launch Instance**.

### Step 6: Connect & Install Java
1.  Wait for instance to be "Running". Click **Connect** → **SSH Client**. Copy the example command.
2.  Connect via your terminal (e.g., `ssh -i "my-key.pem" ec2-user@<PUBLIC-IP>`).
3.  Switch to root user:
    ```bash
    sudo -i
    ```
4.  Update the server:
    ```bash
    dnf update -y
    ```
5.  Install Java 11:
    ```bash
    dnf install java-11-openjdk -y
    ```
6.  Verify Java:
    ```bash
    java -version
    ```

### Step 7: Install MariaDB Client
*We need this to run SQL commands on the remote RDS from the EC2 terminal.*
```bash
dnf install mariadb105-server -y
# Note: On Amazon Linux 2023, you might just need 'mariadb105' or 'mariadb'
dnf install mariadb105 -y
```

## 🔹 PHASE 3: TOMCAT DEPLOYMENT
*Deploying the application server and downloading your files.*

### Step 8: Install Apache Tomcat
1.  Download Tomcat (Check [Apache Tomcat site](https://tomcat.apache.org/) for the latest link, usually 9 or 10):
    ```bash
    cd /opt
    wget https://archive.apache.org/dist/tomcat/tomcat-9/v9.0.86/bin/apache-tomcat-9.0.86.tar.gz
    ```
2.  Extract the file:
    ```bash
    tar -xzvf apache-tomcat-9.0.86.tar.gz
    mv apache-tomcat-9.0.86 tomcat
    ```

### Step 9: Download Artifacts from S3
1.  Go to the `webapps` folder:
    ```bash
    cd /opt/tomcat/webapps
    ```
2.  Download `Student.war` using the link you saved in Step 2:
    ```bash
    wget <PASTE_S3_OBJECT_URL_FOR_STUDENT.WAR>
    ```
3.  Go to the `lib` folder:
    ```bash
    cd /opt/tomcat/lib
    ```
4.  Download `mysql-connector-java.jar`:
    ```bash
    wget <PASTE_S3_OBJECT_URL_FOR_JAR>
    ```

### Step 10: Configure Database Connection (context.xml)
1.  Edit the Tomcat context file:
    ```bash
    cd /opt/tomcat/conf
    vim context.xml
    ```
2.  Press **`i`** to insert. Paste the following **inside** the `<Context>` tag (replace `<RDS-ENDPOINT>` and password with your actual details):

    ```xml
    <Resource name="jdbc/TestDB" 
              auth="Container"
              type="javax.sql.DataSource"
              maxTotal="20" maxIdle="10" maxWaitMillis="-1"
              username="admin" 
              password="admin1234"
              driverClassName="org.mariadb.jdbc.Driver"
              url="jdbc:mariadb://<RDS-ENDPOINT>:3306/studentapp"/>
    ```
3.  Press **`ESC`**, type **`:wq`**, and hit **Enter** to save.

## 🔹 PHASE 4: DATABASE INITIALIZATION
*Setting up the tables.*

### Step 11: Create Database Schema
1.  From your EC2 terminal, connect to the RDS database:
    ```bash
    mysql -u admin -h <RDS-ENDPOINT> -p
    # Enter password: admin1234
    ```
2.  Run the SQL commands:
    ```sql
    CREATE DATABASE studentapp;
    USE studentapp;
    
    -- Paste your table creation SQL here (Example):
    CREATE TABLE Students (
        id INT NOT NULL AUTO_INCREMENT,
        name VARCHAR(255) NOT NULL,
        PRIMARY KEY (id)
    );
    ```
3.  Verify:
    ```sql
    SHOW TABLES;
    DESC Students;
    EXIT;
    ```

### Step 12: Start Tomcat
1.  Start the Tomcat service:
    ```bash
    cd /opt/tomcat/bin
    ./startup.sh
    ```
2.  Check logs to ensure it started without errors:
    ```bash
    tail -f /opt/tomcat/logs/catalina.out
    ```
    *(Press Ctrl+C to exit logs)*.

### 🔍 Troubleshooting
*   **Blank Page / 404:** Check if `Student.war` expanded into a `Student` folder inside `/opt/tomcat/webapps`. (`ls -l`).
*   **Database Connection Error:** Check `/opt/tomcat/logs/catalina.out`. Ensure the password in `context.xml` matches the RDS password.
*   **Cannot Access URL:** Ensure the EC2 Security Group allows **Port 8080** from your IP.
